# SYSTEM RULES

Before implementing anything:

1. Analyze the repository.
2. Identify relevant modules.
3. Describe the implementation plan.
4. Only then modify code.

Never modify working code without clear justification.

Always create automated tests.

After implementation:

1. Run tests
2. Fix failures
3. Ensure no regressions

Every prompt must produce:

- implementation
- tests
- verification
- commit message
