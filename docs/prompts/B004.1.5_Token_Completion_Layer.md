B004_1_5_TOKEN_COMPLETION

Contexto:
A app já usa design tokens, mas faltam tokens semânticos essenciais.
Atualmente usa-se emerald, amber, slate diretamente, quebrando consistência.

Objetivo:
Completar o design system para eliminar completamente cores hardcoded.

---

## 1. Adicionar tokens novos

Em todos os temas:

--color-success
--color-success-foreground

--color-warning
--color-warning-foreground

--color-info
--color-info-foreground

--color-surface-raised
--color-surface-overlay

---

## 2. Atualizar tailwind.config

success, warning, info, surface

---

## 3. Substituir usos diretos

emerald → success  
amber → warning  
slate → muted ou surface  
red → destructive  

---

## 4. Corrigir <pre> blocks

ANTES:
bg-slate-100 text-slate-800

DEPOIS:
bg-muted text-foreground

---

## 5. NÃO alterar lógica

---

## 6. Testes

- Script `web-app/scripts/verify-tokens.js`:
  - Verifica ausência de cores hardcoded (slate, emerald, amber, sky, orange, violet) em .tsx/.jsx
  - Verifica que todos os temas têm os 8 tokens novos
  - Verifica que tailwind.config.js tem success, warning, info, surface
- Executar: `npm run verify-tokens`

---

## 7. Critério de sucesso

- zero cores hardcoded
- todos os temas funcionam corretamente
- contraste consistente
- `npm run verify-tokens` passa
- `npm run build` passa
